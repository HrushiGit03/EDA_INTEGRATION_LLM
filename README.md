# EDA Automation with Ollama LLM + Gradio

## Overview
This repository provides scripts to automate Exploratory Data Analysis (EDA) using an LLM (Ollama) and (optionally) a Gradio UI. The primary utilities are:

- `app.py` — (optional) Gradio web UI to interactively run EDA with the LLM backend (if present).
- `code.py` — Scripted/CLI utilities to run automated EDA tasks non-interactively.

## Files
- `app.py` — Gradio app that accepts a dataset and prompts the Ollama model to produce EDA insights and visualizations (if included).
- `code.py` — Command-line EDA runner that loads a dataset, produces summary statistics, charts, and (optionally) queries the Ollama model for natural-language insights.
- `eda_analysis_utils.py` — Helper functions for loading data, plotting, and preparing prompts.

(If `app.py` is not present, adapt the instructions below to your actual web app implementation.)

## Requirements
- Python 3.8+
- A running Ollama instance (local or remote). See https://ollama.com/docs/ for install and usage.
- Recommended Python packages (install via `pip install -r requirements.txt`):
  - gradio
  - pandas
  - numpy
  - matplotlib
  - seaborn
  - requests
  - python-dotenv

Example `requirements.txt` content:
```
gradio
pandas
numpy
matplotlib
seaborn
requests
python-dotenv
```

## Setup
1. Create and activate a virtual environment:

```bash
python -m venv .venv
# Windows (Command Prompt)
.\.venv\Scripts\activate.bat
```

2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Install and run Ollama (if using a local Ollama model): follow the instructions at https://ollama.com/docs/ and ensure the Ollama daemon is running.

4. (Optional) Create a `.env` file if your code expects environment variables such as `OLLAMA_HOST` or API keys. Example:

```
OLLAMA_HOST=http://localhost:11434
```

## Usage
- Run the Gradio web UI:

```cmd
python app.py
```

Open the displayed Gradio URL in your browser, upload or point to a dataset, and follow the UI to run automated EDA with the Ollama LLM.

- Run `code1.py` for CLI EDA (example):

```cmd
python code.py --data data/my_data.csv
```

(Adjust CLI flags to match `code1.py`'s implementation. If `code1.py` has no CLI parsing yet, run it directly: `python code1.py`.)

## Output
Typical outputs include:
- Text summaries and insights generated by the LLM.
- Saved plots (PNG) and interactive charts (if enabled in the UI).
- A small EDA report (CSV/HTML) summarizing key metrics and anomalies.
